## Overview

The health check endpoint provides comprehensive system status information without requiring authentication. Use this endpoint to:

- ‚úÖ Verify the API is running and accessible
- üîß Check environment configuration
- üîç Validate required services are available
- üìä Monitor system health for CI/CD pipelines

<Info>
This endpoint does **not** require authentication and can be called from anywhere.
</Info>

## Request

<ParamField query="format" type="string" optional>
  Response format preference. Defaults to `json`.
  - `json` - Standard JSON response
  - `plain` - Plain text status only
</ParamField>

## Response

<ResponseField name="status" type="string" required>
  Overall system health status
  - `healthy` - All systems operational
  - `degraded` - Some services may be slow
  - `unhealthy` - Critical issues detected
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  Current server timestamp in ISO format
</ResponseField>

<ResponseField name="version" type="string" required>
  Current API version for compatibility checking
</ResponseField>

<ResponseField name="environment" type="object" required>
  Environment configuration status (no sensitive data exposed)
  
  <Expandable title="environment properties">
    <ResponseField name="APP_ENV" type="string">
      Application environment (production, staging, development)
    </ResponseField>
    <ResponseField name="PORT" type="string">
      Server port number
    </ResponseField>
    <ResponseField name="has_clerk_key" type="boolean">
      Whether Clerk authentication is configured
    </ResponseField>
    <ResponseField name="has_supabase_url" type="boolean">
      Whether Supabase connection is configured
    </ResponseField>
    <ResponseField name="has_database_url" type="boolean">
      Whether database connection is configured
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

<AccordionGroup>
  <Accordion title="üîç Monitoring & Alerting">
    Set up automated monitoring to check API health:
    
    ```bash

    STATUS=$(curl -s https://routiq-backend-prod.up.railway.app/health | jq -r '.status')

      echo "üö® API is $STATUS - alerting team"
    ```

  </Accordion>

  <Accordion title="üöÄ CI/CD Pipeline Validation">
    Validate deployments before routing traffic:
    
    ```yaml

      run: |
        curl -f https://routiq-backend-prod.up.railway.app/health
        if [ $? -eq 0 ]; then
          echo "‚úÖ Deployment healthy"
        else
          echo "‚ùå Deployment failed health check"
          exit 1
        fi
    ```
  </Accordion>

  <Accordion title="üîß Development Setup Verification">

    ```javascript
      }
    ```

## Status Interpretation

| Status | Description | Action Required |
|--------|-------------|-----------------|
| `healthy` | All systems operational | ‚úÖ None |
| `degraded` | Some services slow/unavailable | ‚ö†Ô∏è Monitor closely |
| `unhealthy` | Critical issues detected | üö® Immediate attention |

<Tip>
**Best Practice**: Call this endpoint before making other API requests to ensure the system is available and properly configured.
</Tip>

<Warning>
**Environment Data**: The `environment` object only shows configuration status (true/false) and never exposes actual API keys or sensitive values.
</Warning>
