## Base URL

All API requests should be made to:

```
https://routiq-backend-prod.up.railway.app
```

## API Version

Current API version: **v2.0.0**

All endpoints are prefixed with `/api/v1/` for the current stable version.

## Authentication

<Info>
All endpoints require authentication except `/` and `/health`. See the [Authentication Guide](/authentication) for details.
</Info>

## Request Format

- **Content-Type**: `application/json`
- **Authorization**: `Bearer <jwt-token>`
- **Organization**: `x-organization-id: <org-id>`

## Response Format

All responses follow a consistent JSON structure:

### Success Response
```json
{
  "data": { /* Response data */ },
  "status": "success",
  "timestamp": "2025-01-15T10:30:00.123456Z"
}
```

### Error Response
```json
{
  "error": "Error type",
  "detail": "Detailed error message",
  "timestamp": "2025-01-15T10:30:00.123456Z"
}
```

## Rate Limits

- **Standard**: 1000 requests per hour per organization
- **Sync Operations**: 10 concurrent syncs per organization
- **Burst**: Up to 100 requests per minute

## Status Codes

| Code | Status | Description |
|------|--------|-------------|
| 200 | OK | Request successful |
| 201 | Created | Resource created successfully |
| 400 | Bad Request | Invalid request parameters |
| 401 | Unauthorized | Authentication required |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 422 | Validation Error | Request validation failed |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
