## Overview

This quickstart guide will get you making your first successful API call to Routiq in just a few minutes. We'll walk through authentication, organization setup, and your first patient data request.

<Info>
  **Prerequisites**: You'll need a Routiq account and organization ID. If you don't have these yet, [sign up here](https://app.routiq.com/signup).
</Info>

## Step 1: Check API Status

First, let's verify the API is running and accessible:

<CodeGroup>

```bash cURL
curl https://routiq-backend-prod.up.railway.app/health
```

```javascript JavaScript
const response = await fetch('https://routiq-backend-prod.up.railway.app/health');
const health = await response.json();
console.log(health);
```

```python Python
import requests

response = requests.get('https://routiq-backend-prod.up.railway.app/health')
health = response.json()
print(health)
```

</CodeGroup>

You should see a response like this:

```json
{
  "status": "healthy",
  "timestamp": "2025-01-15T10:30:00.123456",
  "version": "2.0.0",
  "environment": {
    "APP_ENV": "production",
    "PORT": "8000",
    "has_clerk_key": true,
    "has_supabase_url": true,
    "has_database_url": true
  }
}
```

<Check>**API is healthy!** You're ready for the next step.</Check>

## Step 2: Get Your Authentication Token

Routiq uses **Clerk JWT authentication**. You'll get this token automatically when you log into your Routiq frontend application.

<Tabs>
  <Tab title="Frontend Integration">
    If you're building a frontend app, the JWT token is managed automatically:

    ```javascript
    import { useAuth } from '@clerk/clerk-react';

    function MyComponent() {
      const { getToken } = useAuth();
      
      const callAPI = async () => {
        const token = await getToken();
        // Use this token in your API calls
      };
    }
    ```
  </Tab>
  
  <Tab title="Direct Testing">
    For testing, you can get your token from the browser:
    
    1. Log into your Routiq dashboard
    2. Open browser developer tools (F12)
    3. Go to Application/Storage â†’ Local Storage
    4. Look for your Clerk session token
    
    <Warning>Never hardcode tokens in production code!</Warning>
  </Tab>
</Tabs>

## Step 3: Make Your First Authenticated Request

Now let's verify your authentication works:

<CodeGroup>

```bash cURL
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "x-organization-id: YOUR_ORG_ID" \
     https://routiq-backend-prod.up.railway.app/api/v1/auth/verify
```

```javascript JavaScript
const response = await fetch('https://routiq-backend-prod.up.railway.app/api/v1/auth/verify', {
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-organization-id': 'YOUR_ORG_ID'
  }
});

const auth = await response.json();
console.log(auth);
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-organization-id': 'YOUR_ORG_ID'
}

response = requests.get(
    'https://routiq-backend-prod.up.railway.app/api/v1/auth/verify',
    headers=headers
)

auth = response.json()
print(auth)
```

</CodeGroup>

**Expected Response:**
```json
{
  "authenticated": true,
  "organization_id": "org_2xwHiNrj68eaRUlX10anlXGvzX7",
  "message": "Authentication successful"
}
```

<Check>**Authentication successful!** You're now ready to access patient data.</Check>

## Step 4: Get Your Dashboard Data

Let's fetch your practice's dashboard data:

<CodeGroup>

```bash cURL
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "x-organization-id: YOUR_ORG_ID" \
     https://routiq-backend-prod.up.railway.app/api/v1/sync/dashboard/YOUR_ORG_ID
```

```javascript JavaScript
const orgId = 'YOUR_ORG_ID';

const response = await fetch(`https://routiq-backend-prod.up.railway.app/api/v1/sync/dashboard/${orgId}`, {
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-organization-id': orgId
  }
});

const dashboard = await response.json();
console.log('Total patients:', dashboard.patient_stats.total_patients);
console.log('Active patients:', dashboard.patient_stats.active_patients);
```

```python Python
org_id = 'YOUR_ORG_ID'

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-organization-id': org_id
}

response = requests.get(
    f'https://routiq-backend-prod.up.railway.app/api/v1/sync/dashboard/{org_id}',
    headers=headers
)

dashboard = response.json()
print(f"Total patients: {dashboard['patient_stats']['total_patients']}")
print(f"Active patients: {dashboard['patient_stats']['active_patients']}")
```

</CodeGroup>

**Expected Response:**
```json
{
  "organization_id": "org_2xwHiNrj68eaRUlX10anlXGvzX7",
  "patient_stats": {
    "total_patients": 647,
    "active_patients": 47,
    "patients_with_upcoming": 23,
    "patients_with_recent": 31,
    "last_sync_time": "2025-01-15T10:30:00.123456+00:00"
  },
  "current_sync": null,
  "last_sync": {
    "status": "completed",
    "started_at": "2025-01-15T09:15:00.000000+00:00",
    "completed_at": "2025-01-15T09:18:30.000000+00:00",
    "records_success": 647
  },
  "sync_available": true
}
```

<Success>**Congratulations!** You've successfully made your first API call and retrieved patient data.</Success>

## Step 5: Start a Patient Sync (Optional)

If you want to sync fresh data from your practice management system:

<CodeGroup>

```bash cURL
curl -X POST \
     -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "x-organization-id: YOUR_ORG_ID" \
     "https://routiq-backend-prod.up.railway.app/api/v1/sync/start/YOUR_ORG_ID?sync_mode=active"
```

```javascript JavaScript
const orgId = 'YOUR_ORG_ID';

const response = await fetch(`https://routiq-backend-prod.up.railway.app/api/v1/sync/start/${orgId}?sync_mode=active`, {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-organization-id': orgId
  }
});

const sync = await response.json();
console.log('Sync started:', sync.sync_id);

// Poll for progress
const checkProgress = async (syncId) => {
  const statusResponse = await fetch(`https://routiq-backend-prod.up.railway.app/api/v1/sync/status/${syncId}`);
  const status = await statusResponse.json();
  console.log(`Progress: ${status.progress_percentage}% - ${status.current_step}`);
};
```

</CodeGroup>

## Common Issues

<AccordionGroup>
  <Accordion title="401 Unauthorized Error">
    **Problem**: Your JWT token is invalid or expired.
    
    **Solution**: 
    - Verify your token is correctly formatted
    - Check if you need to refresh your session
    - Ensure you're using the latest token from Clerk
  </Accordion>

  <Accordion title="403 Forbidden Error">
    **Problem**: Your token is valid but you don't have access to the organization.
    
    **Solution**:
    - Verify your organization ID is correct
    - Check that your account has access to this organization
    - Contact support if you believe this is an error
  </Accordion>

  <Accordion title="Empty Patient Data">
    **Problem**: Dashboard shows 0 patients or no sync data.
    
    **Solution**:
    - Run a sync first: `POST /api/v1/sync/start/{org_id}`
    - Check if your Cliniko credentials are configured
    - Verify your practice management system has patient data
  </Accordion>
</AccordionGroup>
